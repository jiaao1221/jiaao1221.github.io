发表于 2018-11-23 | 更新于 2018-11-27 | 分类于 学习笔记 | 阅读次数：

前言

这个部分感觉是MyBatis里比较复杂的一块，也是很重要的一块，如果弄清楚了，那么对复杂映射背后的原理理解将会更加深刻。
我们还是延用之前搭的demo继续代码调试。

ResultSetHandle接口

先看看接口的源码：

只有两个方法。通过传参可以看出来，handleResultSets是处理普通sql语句的操作，handleOutputParameters是处理存储过程的结果。该接口只有一个默认实现：DefaultResultSetHandler，其实现的步骤就是将Statement执行后的结果集，按照Mapper文件中配置的ResultType或ResultMap来封装成对应的对象，最后将封装的对象返回。

这里会遇到DefaultResultHandler，因为resultHandler为空，所以这里生成了一个DefaultResultHandler对象。首先看一下resultHandler，这个接口是用来处理结果集，如果没有自定义的resultHandler，就会使用DefaultResultHandler，先来看一下它的实现：

这个接口其实就只需要实现一个方法handleResult(),其中参数ResultContext包含了结果，我们可以对其进行处理。例如DefaultResultHandler的处理方法就是将结果加入到list中,最后加入到multipleResults中。我们也可以自定义resultHandler进行处理。有了ResultHandler，进入handleRowValues()方法：

所有的结果将存到resultContext中。skipRows()起到翻页的作用。resolveDiscriminatedResultMap如果有定义鉴定器的话选择合适ResultMap。rowValue就是映射后得到的结果，这里是User。storeObject是把结果装载到resultContext，再把resultContext交给resultHandler处理。现在进入getRowValue()方法：

这里有两个方法比较重要，一个是createResultObject()，通过反射得到User实例，一个是applyAutomaticMappings()，因为在配置文件默认就是自动匹配，会将字段名相同的数据填入。
先看createResultObject()：

这里有两个重要的地方，一个是createResultObject()方法，一个是return this.configuration.getProxyFactory().createProxy(resultObject, lazyLoader, this.configuration, this.objectFactory, constructorArgTypes, constructorArgs);得到懒加载的动态代理对象。这里不讲懒加载的机制，只看createResultObject()方法。constructorArgTypes 和constructorArgs 是如果定义了有参数的构造函数的参数和类型。

这里由于是简单映射，没有定义带参的构造函数，typeHandler等，直接跳到最后this.objectFactory.create(resultType)，通过resultType反射得到User实例。跳出后回到上一层的createResultObject，之后就是懒加载的判断，于是再跳出回到最开始的getRowValue(),开始为User注入属性。
进入applyAutomaticMappings():

我们直接看这两句：
TypeHandler<?> typeHandler = rsw.getTypeHandler(propertyType, columnName); value = typeHandler.getResult(rsw.getResultSet(), columnName);
先得到用来列处理的typeHandler，这个typeHandler可以是自己定义注册的。对于简单地整型，String等就不需要自定义。首先处理的行是ID，这里的typeHandler就是LongtypeHandler,通过getResult得到ID列的值，这里就要深入到JDBC与MyBatis的动态代理，之后再讨论。
现在得到value后要把值赋予给User,这里是通过metaObject对User进行赋值。MetaObject是Mybatis提供的一个用于方便、优雅访问对象属性的对象，通过它可以简化代码、不需要try/catch各种reflect异常，同时它支持对JavaBean、Collection、Map三种类型对象的操作。通过metaObject.setValue设置值。一个while循环完之后，这条记录就结束了，一个完整的User实例也生成了。

结尾

复杂映射包括一对一，一对多，多对多，其实走的流程和之前差不多，如果某一个resultMap开启了懒加载，则会对结果对象做一个动态代理，将原本的get()/set()方法代理成一个向数据库查询的方法，一旦懒加载执行，之后再次执行就不会向数据库再次提交。
